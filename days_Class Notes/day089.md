1. 요구사항 정의서

안녕하십니까 제가 담당한 기능과 게시판들에 대해 설명을 드리겠습니다.

제가 담당한 부분은 크게 세 가지로 나눌 수 있습니다.
첫 번째는 사용자가 관리자에게 문의할 수 있는 고객센터 기능,
두 번째는 다양한 필터를 제공하는 사료 검색 게시판,
마지막으로 기존 게시판과 신규 담당 게시판의 API 도입입니다.

요구사항 정의서 기준으로 대부분의 기능은 완료되었지만
일부 API 기능은 현실적인 비용 문제로 보류했습니다.



2. 화면설계서
다음 화면 설계서는 제가 담당한 게시판을 한 화면에 담은 것입니다.
기존 게시판은 API 중심, 신규 게시판은 전체적인 UI 중심으로 정리하였습니다.



3. 시연영상

시연 영상을 통해 자세한 설명을 드리겠습니다.

고객센터는 상단의 메뉴바를 통해 진입할 수 있습니다.
여기서 자주 묻는 질문을 확인하고 1:1 문의를 남길 수 있습니다.
작성한 질문과 답변 상황은 우측의 내 질문 보기 버튼이나
마이페이지를 통해서 확인이 가능합니다.

실시간 상담이 필요한 경우
하단의 채팅 상담 버튼을 통해 채널톡 API와 연결됩니다.
이용자가 문의를 입력하면 관리자에게 채팅 알림이 전달되고
담당자를 배정해 실시간 상담을 진행할 수 있습니다.	
인사말과 주요 응답은 매크로를 활용해
효율적인 상담이 가능하도록 구성했습니다.

이렇게 상담이 마무리됩니다.

관리자의 경우 활성화 버튼이 노출됩니다.
이 버튼으로 사용 여부와 유저 노출 여부를 결정할 수 있습니다.
그리고 답변을 작성할 수 있는데 데이터 관리 차원에서 수정이 불가능하도록 만들었습니다.
답변을 작성하면 자동으로 답변 완료 처리가 됩니다.
그리고 제목, 내용, 유저로 검색이 가능합니다.
답변 여부에 따른 정렬도 가능하도록 하였습니다.


사료 찾기 게시판에서는 데이터 전체 출력을 방지하기 위해 검색 조건은 1개 이상 선택하도록 설정하였습니다.
5개 단위 페이징 기능과 이름, 브랜드, 별점 순의 다양한 정렬 조건을 설정하였습니다.
상세 검색 버튼을 클릭하면 더 다양한 조건을 설정할 수 있습니다.
이렇게 나온 결과를 통해 자세한 사료의 정보를 확인하고 리뷰 게시판과 연동하여 작성된 리뷰를 확인할 수 있습니다.
혹시 검색 조건 설정이 어렵다면 ai에게 도움을 요청할 수 있습니다.
원하는 조건을 설명하고 질문하기 버튼을 누르면 ai가 자동으로 필터를 조정하고 이유를 설명해 줍니다

마지막으로 기존 게시판에 도입된 api 2가지를 소개해 드리겠습니다.
첫 번째는 리뷰 작성 게시판에 도입된 ai입니다.
사용자가 리뷰를 작성하고 다듬기 버튼을 누르면 좀 더 정중한 표현을 제안합니다.
사용자는 비교 후 적용 여부를 선택할 수 있습니다.

다음은 사료 등록 게시판에 도입된 api입니다.
사용자가 이미지로 된 라벨 데이터를 첨부하면 api가 자동으로 텍스트를 추출해 좀 더 쉽게 자료를 입력할 수 있습니다.
