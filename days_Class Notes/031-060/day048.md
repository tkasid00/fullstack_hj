 1. 모델링의 3대 특징
  - 추상화
  - 단순화
  - 정확화
  ---------
  no animal (고양이-페르시안)  
  ---------
  1  cat
  2  dog

 2. 모델링 
  - 설명+분석+설계
  - 단계 
     1) 개념(추상) :animal
     2) 논리(정의) : name, age
     3) 물리(저장소) : <table> name varchar2(20)

 3. ANSI-SPARC 스키마(구조적 단계)
  - 외/사 -> 외부 스키마/사용자가 보는 관점(개인, 메뉴판)
  - 개/조 -> 개념 스키마/설계도, 관계도(조직 전체 관점, 레시피)
  - 내/저 -> 내부 스키마/저장소(음식, 하드디스크)   
 
 4. erd
  - 1:N 관계 : <고객> ―++――――――ㅇ∈<주문>
  - 피터 첸
  - 도관이차 -> 도형 배치 - 관계 설정 - 관계명(이름) 기술 - 관계차수 기술
  - 구조, 흐름에 따라
 
 5. 엔티티
  - 기본 엔티티(primary key) : 독립적, 고유 식별자
  - 중심 엔티티(main) : 전체 모델의 중심, 식별자 무관
  - 행위 엔티티 : 이벤트, 활동 중심
  - 개념 엔티티 : 추상적 개념

 6. 속성
  - 기본 속성(입력값) : 원금, 이자율 -> 직접  지정
  - 파생 속성(계산값) : 총 금액 -> 계산식 거침(원금*이자율=총 금액)

 7. 데이터모델의 개념  
  - 도메인 : 값의 허용 범위+형식(PNAME VARCHAR2(20))
  - 시스템 카탈로그 : DB의 구조와 정보 저장
  - 용어 사전 : 용어 정의
  - 속성 사전 : 속성 목록

 8. ERD vs UML
  - ERD : 단일 실선
  - UML : 실선-연관관계, 점선-의존관계(방향성, 다중성 표현)
 
 9. 관계 
  - 존재(소속)/행위(배송근거)
  - 표기법 : 관계명, 관계차수(기수성)
  - 관계 도출 
        1) 규칙 : 엔티티 사이 연관규칙
        2) 조합 : 정보 조합
        3) 설명 : 규칙 서술
        4) 동사 : 동사로 관계 연결(고객이 주문한다)

 10. 주식별자(PK)        
  - 유일성, 최소성(중복x, NOT NULL)  

    ※ [KEY] - day046    
    1. KEY : 튜플, 행, 레코드(1, 도비, 10)들을 구분할 수 있는 기준 필드(열, ATTRIBUTE)
    2. 종류 : 후보키, 대체키, 슈퍼키(복합키), 외래키
    3. 후보키 : 기본키로 사용 가능한 속성(민증, 학생증, 사원증) 
                -> 유일성O, 최소성O
    4. 기본키 : 후보키 중 선택된 주요 키(PRIMARY KEY), 중복X/NOT NULL
    5. 대체키 : 후보키 중 기본키를 제외한 나머지 키
    6. 슈퍼키(복합키) : 속성+속성으로 구성된 키
                        -> 유일성O, 최소성X
    7. 외래키 : 테이블(ENTITY)과 테이블 연결 속성

 11. 인조식별자
  - 시스템이 자동 생성해주는 값(외래키 이름을 지정하지 않아도 자동으로 생성해줌)

 12. 비식별자
  - 식별자 전달x, 자식이 독립적, 복잡성 방지
 
 13. 성능 튜닝 : SQL+모델링
  - 모델링 순서 : 정용트반조검
        1) 정규화
        2) 용량 산정
        3) 트랜젝션 유형 : 한 번에 처리 -> 상세보기(조회수 올리고 페이지 보기)
        4) 반정규화
        5) 조정 : PK/FK 등 모델링 조정
        6) 검증 : 성능 검증, 테스트
 
 14. 반정규화 
  - 중복 컬럼, 파생 컬럼, 기능 등

 15. 로우체이닝 
  - 하나의 행이 여러 테이블 나눠서 저장(많은 컬럼)

 16. 파티셔닝
  - 물리적 분할, 성능 향상  

 17. FK
  - 관계 유지
  - 조인 성능을 위해 INDEX 생성
  - 제약조건을 생성해도 INDEX는 필요

 18. 분단데이터베이스
  - 마스터데이터 복제, 실시간 업무, 백업 사이트 구성
  - GSI(Global Single Instence) : 단일 인스턴스, 통합 운영 -> 분산과 반대
