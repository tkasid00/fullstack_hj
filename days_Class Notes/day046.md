#2. 정규화(NF:Noraml Form)
1. 이상 제거 
2. 이상 - 공간 낭비, 삽입, 갱신, 삭제 이상
3. 1FN~5NF

#3. 1NF(원자값)
- 원자 : 더 이상 쪼갤 수 없는 단위
- 각 속성은 하나의 값만 가짐
    ex) STD_ID      SNAME       GRADE
        1111        BUTTER      A+, A
               ▽   ▽   ▽   ▽
        STD_ID      SNAME       GRADE
        1111        BUTTER      A+
        1111        BUTTER      A

#4. 2NF(부분함수 종속 제거)        
 1. 종속 관계 : X(결정자)를 알면 Y(종속자)를 알 수 있다
 2. 모든 속성이 기본 키에 완전함수종속 -> 제2정규형
 3. 종속자가 기본키에만 종속
    ex) <학번, 수강 과목> -> 성적

|1-2 |학번              | 학부        |

            종속자가 기본키를 구성하는 속성 중 일부에게만 종속

#5. 3NF(이행함수 종속 제거)                
1. X -> Y-> Z
2. 다른 후보키에 의존하지 않음
3. 테스트 시 INSERT, UNPDATE, DELETE 이상 확인 
    ex) <학번:PK>을 알면 <지도교수:후보키>를 알고
                       <지도교수:후보키>를 알면 <학부>를 알 수 있다
        -> [학번(PK), 지도교수], [지도교수(PK), 학부]              

<img src="/fullstack_hj/track002_web_basic/img/erd/nf.png"> 


  ※ [KEY]
    1. KEY : 튜플, 행, 레코드(1, 도비, 10)들을 구분할 수 있는 기준 필드(열, ATTRIBUTE)
    2. 종류 : 후보키, 대체키, 슈퍼키(복합키), 외래키
    3. 후보키 : 기본키로 사용 가능한 속성(민증, 학생증, 사원증) 
                -> 유일성O, 최소성O
    4. 기본키 : 후보키 중 선택된 주요 키(PRIMARY KEY), 중복X/NOT NULL
    5. 대체키 : 후보키 중 기본키를 제외한 나머지 키
    6. 슈퍼키(복합키) : 속성+속성으로 구성된 키
                        -> 유일성O, 최소성X
    7. 외래키 : 테이블(ENTITY)과 테이블 연결 속성

#7. 4NF(다치종속 제거)
    ex) 프로그래밍 과목에 담당교수2, 교재코드2
    <과목, 교수코드, 교재코드>
    2*2=4, 2*3=6....
      ▽   ▽   ▽   ▽
    <과목, 교수코드>    2
    <과목, 교재코드>    2

#8. 5NF(조인종속 제거)
 - 하나의 테이블을 여러 개로 분해 후 다시 조인했을 때 데이터 손실은 없지만 불필요한 데이터가 생김 
 - N:M 다대다 관계에서 사용 -> 중간 테이블 생성(많이 쓰는 필드)
    ex) 교수는 여러 개의 교재를 갖는다 / 교재는 여러 명의 교수가 사용할 수 있다
        <교수> ∋-----------------------------∈ <교재>
                      ▽   ▽   ▽   ▽
        <교수> +-------∋ <교수교재> ∈ -------+ <교재>
