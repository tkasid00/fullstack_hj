*SQLD

1. 명령어 분류
    1) DDL(정의어) - CADR [CREATE, ALTER, DROP, RENAME]
    2) DML(조작어) - CRUD [INSERT, SELECT, UPDATE, DELETE]
    3) DCL(제어어-권한) - RG [GRANT, REVOKE]
    4) TCL(제어어-트랜젝션) -CR [COMMIT, ROLLBACK, SAVEPOINT]

2. PRIMARY KEY 설정 방법
    1) CREATE - 컬럼명   자료형  PRIMARY KEY 
    2) CREATE - 하단에 CONSTRAINT 테이블명_PK PRKMARY KEY(컬럼명)
    3) ALTER TABLE 테이블명 ADD CONSTRAINT 테이블명_PK PRKMARY KEY(컬럼명)

3. 인덱스 설정 방법
    - CREATE INDEX 인덱스테이블명 ON 원본테이블명(컬럼명);    

4. 제약조건
    - CHECK : 특정 컬럼 무결성 유지    

5. 고립성
    - Dirty Read : COMMIT되지 않은 데이터 읽기
    - Non-Repeatable Read : 결과를 다르게 읽기
    - Phantom Read : 첫 번쨰 쿼리에는 없던 유렁 레코드가 두 번째 쿼리에서 나오기
