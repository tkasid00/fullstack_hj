
# 🐾 최종 브랜드 4개 제안 (개·고양이 공용 설계)

## **1. 퓨어테일(PureTail) – 미국, 프리미엄 기능성 브랜드**

**성향:**

* 고단백, 저인(특히 신장 관리 라인 강함)
* 연어·칠면조 같은 고품질 단백질 중심
* 강아지·고양이 모두 “프리미엄 기능성” 포지션

**포트폴리오 장점:**

* 처방식(신장·체중 관리·위장 케어) 라인 넣기에 최적
* 네 영양소 범위(NUTRIENTRANGE)와 연동하기 아주 좋음

**한 줄 정체성:**

> “프리미엄 단백질과 기능성 영양 균형에 집중한 브랜드”

---

## **2. 내추럴포레스트(NaturalForest) – 캐나다, 자연식 기반 브랜드**

**성향:**

* 자연식 성분, 알레르기 최소화
* 고구마·호박·오리·양고기 같은 재료를 많이 사용
* 시니어/인도어 라인 개발하기 좋음

**포트폴리오 장점:**

* “민감한 위장/저알러지” 라인업 만들기 좋다
* 강아지/고양이 공용 레시피도 자연스럽게 가능

**한 줄 정체성:**

> “자연에서 온 재료로 민감한 아이들도 편하게 먹는 브랜드”

---

## **3. 밥튼튼(BopTenTen) – 한국, 중가형 데일리 사료 브랜드**

**성향:**

* 일상 급여용, 비교적 저렴
* 닭/참치/연어 같은 흔한 단백질 기반
* 기능성은 약하지만 ‘가성비’ 컨셉

**포트폴리오 장점:**

* 데이터량 늘리기 좋고, 일반 라인을 다수 배치하기 좋다
* 브랜드 간 가격 차이를 설명하기에도 좋음

**한 줄 정체성:**

> “일상적인 급여에 맞춘 한국형 데일리 브랜드”

---

## **4. 메디포우(MediPaw) – 독일, 처방식 전문 브랜드**

**성향:**

* 신장·비만·요로·위장 등 각종 처방식
* 구성비율은 엄격하게 설정된 라인
* 너의 ‘처방식’ 기반 추천 알고리즘과 매우 잘 맞음

**포트폴리오 장점:**

* NUTRIENTRANGE 테이블의 저단백/저인/저나트륨보다 정교한 기준을 설명하기 좋다
* “의학적 근거 기반 포뮬러”라는 설정으로 DB에 신뢰도 상승

**한 줄 정체성:**

> “수의학 영양학 기반의 전문 처방식 브랜드”

2. 테이블 생성
CREATE TABLE foodbrand (
    brandid        NUMBER        NOT NULL,
    brandname      VARCHAR2(100) NOT NULL,
    country        VARCHAR2(100),
    brandtype      VARCHAR2(50)  NOT NULL,   
    category       VARCHAR2(50),             
    brandinfo      VARCHAR2(500),           
    CONSTRAINT pk_foodbrand PRIMARY KEY (brandid)
);

CREATE TABLE food (
    foodid          NUMBER          NOT NULL,
    foodname        VARCHAR2(100)   NOT NULL,
    brandid         NUMBER          NOT NULL,
    description     VARCHAR2(500),
    mainingredient  VARCHAR2(200)   NOT NULL,
    subingredient   VARCHAR2(200),
    pettypeid       NUMBER          NOT NULL,   -- 1=고양이, 2=강아지
    category        VARCHAR2(50)    NOT NULL,   -- 처방식/일반/시니어 등
    petagegroup     VARCHAR2(50),               -- 키튼/어덜트/시니어
    isgrainfree     CHAR(1),                    -- Y/N
    calorie         NUMBER(5,1),                -- kcal/100g
    foodtype        VARCHAR2(20),               -- 건식/습식
    foodimg         VARCHAR2(300),              -- 사료 이미지 경로
    CONSTRAINT pk_food PRIMARY KEY (foodid)
);
