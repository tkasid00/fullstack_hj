**A. 화살표 함수**
1. 기본
- 형태 : (인수1, 인수2) => {표현식}
- 인수가 하나인 경우 (괄호) 생략 가능
- 표현식이 하나인 경우 {괄호} 생략 가능 -> 자동 리턴
- 표현식이 여럿인 경우 {괄호} 사용 -> 리턴값 명시 필요

2. 과제
    - 화살표 함수로 변경하기
    ```js
    //문제
    function ask(question, yes, no) {
    if (confirm(question)) yes()
    else no();
    }

    ask(
    "동의하십니까?",
    function() { alert("동의하셨습니다."); },
    function() { alert("취소 버튼을 누르셨습니다."); }
    );

    //답
    ask(
    "동의하십니까?",
    () => alert("동의하셨습니다."),
    () => alert("취소 버튼을 누르셨습니다.")
    );

    ```

**B. 기본 문법 요약**
1. 세미콜론 생략
- 코드 블록({...} )이나 반복문같이 코드 블록과 함께 구성되는 문법은 생략 가능

2. 엄격 모드(use strict) 
- 모던 자바스크립트 기능 활성화
- 스크립트 최상단이나 함수 본문 최상단에 'use strict' 선언하여 활용

3. 자료형
- BigInt : 큰 숫자
- null : 빈 값, 존재하지 않는 값
- undefined : 값이 할당되지 않은 상태
- typeof 예외 : typeof null == "object" // 언어 자체의 오류 <br/>
                typeof function(){} == "function" // 함수는 특별하게 취급

**C. Chrome으로 디버깅하기**
1. 중단점(breakpoint) 
    - 자바스크립트의 실행이 중단되는 코드 내 지점
    - 실행이 중지된 시점에 변수가 어떤 값을 담고 있는지 확인 가능
    - 줄 번호 -> 우클릭 -> 조건부 중단점(conditional breakpoint) 설정 가능
    - 스크립트 내 중단점 설정 방법 : debugger; 코드 삽입

2. 디버깅 패널
    1) Watch 
        - Add Expression 버튼 +를 클릭해 원하는 표현식을 입력한 후 Enter를 누르면 중단 시점의 값을 보여줌 
        - 실행 과정 중 입력한 표현식 재평가 진행

    2) Call Stack 
        - 코드를 해당 중단점으로 안내한 실행 경로를 역순으로 표시
        - 콜 스택 내의 항목 클릭 시 디버거가 해당 코드로 이동, 변수 재평가
    3) Scope 
        - 현재 정의된 모든 변수를 출력
        - Local : 지역변수
        - Global : 전역변수

3. 실행 추적
    1) Resume(F8)
        - 실행을 재개, 추가 중단점이 없는 경우 디버거는 동작x

    2) Step(F9)
        - 다음 명령어 실행, 계속 누를 시 스크립트 전체를 문 단위로 하나하나 실행
        - 함수 내부로 들어가 함수 본문 첫 번째 줄에서 실행을 멈춤
        - 비동기 동작 무시

    3) Step over(F10)
        - 다음 명령어를 실행하되 함수 안으로 들어가진 않음 
        - alert 같은 내장함수에는 해당X, 직접 작성한 함수일 경우 해당
        - 보이지 않는 곳에서 중첩 함수를 실행
        - 함수 호출 시 내부에서 어떤 일이 일어나는지 궁금하지 않을 때 사용
    
    4) Step into(F11)
        - 비동기 동작을 담당하는 코드 진입, 필요할 경우 비동기 동작이 완료될 때까지 대기
    
    5) Step out(Shift+F11)    
        - 현재 실행 중인 함수의 실행을 계속 이어가다가 함수 본문 마지막 줄에서 실행을 멈춤

    6) -/->
        - 모든 중단점을 일시적으로 활성화/비활성화, 실행에는 영향X

    7) (||)
        – 예외 발생 시 코드를 자동 중지시켜주는 기능을 활성화/비활성화
